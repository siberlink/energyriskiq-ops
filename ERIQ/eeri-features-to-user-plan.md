## PART 14 â€” Probability Calculation Schema (Backend)

Probabilities are designed to be:
- **Defensible** â€” "historical conditional tendencies"
- **Stable** â€” not jittery week-to-week
- **Auditable** â€” why the number exists
- **Tier-aware** â€” Free sees none, Personal sees light ranges, Pro sees confidence + persistence

### 1) Core Concept

Compute "next-week probabilities" conditioned on a **Weekly State**:

```
Weekly State =
  EERI band (based on weekly average)
  + EERI trend (rising/stable/falling vs prior week)
  + Divergence regime (confirming/mixed/diverging) [optional but powerful]
  + Optional: "shock week" flag (weekly high â€“ low exceeds threshold)
```

Then ask historically:

> "For past weeks with the same state, what happened to each asset in the following week?"

### 4) Confidence Labeling (Marketable + Honest)

Confidence shown as a simple label tied to:
- Sample size bucket
- Dispersion (how inconsistent outcomes were)

**User-facing labels:**

| Label | Meaning |
|-------|---------|
| **High** | Stable tendency with enough history |
| **Medium** | Decent tendency, some variance |
| **Low** | Mixed outcomes or limited history |

This keeps it credible and avoids "fake precision".
